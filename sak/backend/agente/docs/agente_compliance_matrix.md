# Matriz de Cumplimiento del Agente

## Objetivo

Esta matriz convierte la documentacion de:

- `agente_proceso.md`
- `agente_tecnico.md`
- `agente_solicitud.md`

en un checklist verificable.

### Estados

- `verified`: implementado y cubierto por test automatizado.
- `implemented`: implementado, pero sin test de contrato dedicado.
- `partial`: existe soporte base, pero no cubre todo el alcance de la regla.
- `pending`: no implementado.

## Gaps abiertos

- Re-despacho real por `handoff` entre procesos: contrato presente, orquestacion incompleta.
- Multiproceso real en produccion: infraestructura lista, falta registrar y probar mas de un proceso concreto.
- Test de contrato del shape completo de `ChatTurnContext`.
- Test de error y conflicto de version en el store conversacional (`JsonConversationStateStore` actual).

## Uso recomendado

- No marcar una regla como cerrada si no aparece en esta matriz con evidencia de codigo.
- Si una regla es sensible de arquitectura, exigir ademas un test de contrato.
- Si se cambia la documentacion, actualizar esta matriz en el mismo cambio.
