Analiza un turno donde existe una consulta pendiente para completar un atributo obligatorio.

Objetivo:
- determinar si el mensaje del usuario fue un intento de responder la consulta pendiente
- determinar si el mensaje es independiente y debe volver al flujo normal
- marcar como ambiguo si no puedes decidir con confianza

Reglas:
- usa "consulta_pendiente" como foco principal
- usa "solicitud_actual" solo como contexto
- no intentes editar la solicitud en esta etapa
- si el usuario parece intentar responder el atributo pendiente, usa decision_type "answer_attempt"
- si el usuario esta pidiendo otra accion sobre la solicitud, agregando materiales, o enviando cualquier mensaje que no tenga relacion con el atributo pendiente, usa "independent_message"
- saludos, preguntas generales, comentarios fuera de tema y cualquier mensaje que claramente no sea una respuesta al atributo pendiente son SIEMPRE "independent_message", nunca "ambiguous"
- usa "ambiguous" solo cuando el mensaje podria ser una respuesta al atributo pero no queda claro (por ejemplo: "no se", "tal vez", "depende")
- no devuelvas texto fuera del JSON

Formato:
{
  "decision_type": "answer_attempt" | "independent_message" | "ambiguous",
  "reply_to_user": "texto opcional",
  "confidence": 0.0,
  "warnings": []
}
